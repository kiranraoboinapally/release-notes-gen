## 🚀 Getting Started

If you're looking to get involved, you’re in the right place! Here’s how you can contribute:

### 1. **Fork the repository**
Fork the repo to your own GitHub account:
- Navigate to the [ReleaseNotesGen GitHub repository](https://github.com/kiranraoboinapally/release-notes-gen).
- Click on the **Fork** button at the top right of the page.


### 2. **Clone your forked repository**
After forking the repository, clone it to your local machine:
```bash
git clone https://github.com/kiranraoboinapally/release-notes-gen.git
cd release-notes-gen
```

### 3. **Create a branch for your changes**
We work with branches for each feature or bugfix, so always create a new branch when making changes:
```bash
git checkout -b feature/your-feature-name
```

### 4. **Make your changes**
Make the necessary changes to the codebase or documentation. Be sure to follow the code style and write any tests that are necessary.

### 5. **Commit your changes**
Once your changes are ready, commit them with a meaningful commit message:
```bash
git add .
git commit -m "Your commit message describing the change"
```

### 6. **Push to your fork**
Push your changes to your fork on GitHub:
```bash
git push origin feature/your-feature-name
```

### 7. **Create a Pull Request**
Now, create a pull request from your fork to the main repository. You can do this by:
- Navigating to the "Pull Requests" tab on GitHub.
- Click on "New Pull Request" and select your branch to merge into the `main` branch of the original repository.

Please note that this project adheres to a **Code of Conduct** that we expect all contributors to follow. This ensures that we maintain a friendly and respectful environment for everyone involved.

## 📝 Pull Request Guidelines

When submitting a pull request:
- Provide a clear description of the change, what issue it fixes, and any relevant context.
- If you’re fixing a bug, link to the issue it addresses (e.g., `Fixes #123`).
- Include tests where applicable.
- Follow the existing coding style.

---

## 🐛 Reporting Bugs

If you encounter any bugs or issues, please **open an issue** by navigating to the [Issues section](https://github.com/yourusername/release-notes-gen/issues) and creating a new issue. Please be sure to:
- Describe the bug in detail.
- Include steps to reproduce the issue.
- Share any relevant screenshots or logs.

---

## 📄 Feature Requests

If you have a feature request, please **open an issue** and use the following format:
- **Title**: Feature request: [brief feature description]
- **Description**: Explain what the feature would do, why it’s useful, and how it could be implemented.

---

## 🙋 Frequently Asked Questions

**Q: Do I need to ask before I start working on a feature or bugfix?**

A: It's helpful, but not required. If you're unsure, feel free to check open issues or ask in the discussions.
